# Advanced Features

:::caution Enterprise Only
All features on this page are available exclusively on **Enterprise** plans. [Contact us](mailto:support@permit.io) or [schedule a demo](https://calendly.com/permit-io/demo) to learn more.
:::

## Agent Fingerprinting

`Enterprise`

Verify _what_ an agent actually is — not just what it claims to be. The gateway runs a lightweight behavioral interrogation when an agent connects, building a unique fingerprint from response patterns, timing, and capability signatures. This detects impersonation, credential reuse, and behavioral drift before any tool call is authorized.

## Inject Security Snitch Skills

`Enterprise`

Inject lightweight monitoring capabilities directly into an agent's active session. Instead of only observing traffic at the gateway boundary, snitch skills provide in-session visibility — tracking how an agent chains tool calls, what data it accumulates, and whether its behavior matches its declared purpose.

## Shadow Agent Detection

`Enterprise`

Identify unregistered or unauthorized AI agents connecting to your MCP servers outside of approved channels. Shadow Agent Detection uses behavioral analysis and traffic pattern recognition to find rogue connections — even when they bypass the gateway entirely — and alerts your security team so they can enforce gateway-only routing.

## Human in the Loop (HITL)

`Enterprise`

Add step-up approval gates for high-risk agent actions. When an agent requests a sensitive operation — deleting records, transferring funds, accessing PII — the gateway pauses execution and routes an approval request to a designated human reviewer. Routine operations continue unimpeded; only the actions that matter most require explicit sign-off.

## Intent-Based Access Control

`Enterprise`

Define what each agent is allowed to do by declaring its intended purpose. Instead of authorizing each tool call in isolation, the gateway evaluates the agent's declared intent against the full action sequence — enabling holistic policy checks, conflict detection, and preemptive risk assessment before execution begins.

## Permission Receipts

`Enterprise`

Auditable records of every permission grant — what was approved, when, and by whom. Each receipt captures the granting user, agent, MCP server, tool, trust level, expiry date, and a cryptographic signature, providing a tamper-evident audit trail for compliance and governance. Receipts can be downloaded individually or exported in bulk.

## Time-Limited Consent

`Enterprise`

Set custom consent windows that automatically expire — for example, 2-week access for a contractor engagement. When the window closes, access is revoked without manual cleanup. If the engagement extends, the user simply re-consents.

:::info Built-in session expiry (all plans)
The gateway already enforces a **90-day hard session expiry** and a **30-day soft (inactivity) expiry** on all plans — see [Consent Service: Session expiry](/permit-mcp-gateway/consent-service#session-expiry). This Enterprise feature adds **configurable consent windows** beyond those defaults.
:::
